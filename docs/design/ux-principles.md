# UX Principles

**Status:** Binding  
**Parent:** [Design Bible](README.md)

## First principle

Every pixel must help a better trading decision. Otherwise delete it.

## Ten laws

1. **Workflows, not pages** — design the job (trade, understand, prove, decide).  
2. **The product disappears** — traders think about markets, not chrome.  
3. **Terminal is flagship** — other desks support Terminal.  
4. **Keyboard first, mouse second** — every desk ships a shortcut map (`?`).  
5. **Zero page scroll on OS desks** — Terminal / Book / Research / Counsel use full-bleed shells; panels scroll internally.  
6. **One of each** — one SessionBar, one ticket, one counsel strip per context; no stacked status systems.  
7. **Silence is a feature** — WAIT / empty / offline are first-class (Counsel Silence Protocol).  
8. **Speed over decoration** — motion only when it clarifies state (180–220ms).  
9. **Honest emptiness** — never fabricate to fill space.  
10. **Accessibility is default** — focus, labels, contrast, reduced-motion respect.

## Interaction patterns

| Pattern | Use |
|---|---|
| Command palette | Cross-desk jump (⌘K) |
| OS shortcuts ⌘1–4 | Terminal → Book → Research → Counsel |
| Confirm dialogs | Destructive or capital-moving actions only in Terminal |
| Optimistic UI | Only when rollback is safe and truthful |
| Skeletons | Loading — never fake numbers |

## Density

Institutional density: high information, calm hierarchy, no card spam.  
Cards only when they contain a discrete interaction. Prefer borders and rhythm over shadows.

## Competitive avoidance

Do not copy MT5 / TradingView / crypto exchange / Bloomberg layouts.  
Invent QuantForg workflows (Counsel Strip, Risk DNA, Silence Protocol, Promotion Pipeline, etc.).
